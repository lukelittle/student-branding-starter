# Contributing to Student Branding Starter

Thank you for considering contributing to this project! This template helps students build their personal brand, and your contributions make it better for everyone.

## How to Contribute

### Reporting Bugs

If you find a bug, please open an issue with:

- **Clear title** describing the problem
- **Steps to reproduce** the issue
- **Expected behavior** vs actual behavior
- **Screenshots** if applicable
- **Environment details** (OS, Hugo version, browser)

### Suggesting Enhancements

Have an idea to improve the template? Open an issue with:

- **Clear description** of the enhancement
- **Use case** - why would this be helpful?
- **Examples** from other sites if applicable
- **Implementation ideas** if you have them

### Pull Requests

1. **Fork the repository**
2. **Create a branch** from `main`:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes**
4. **Test locally**:
   ```bash
   hugo server -D
   ```
5. **Commit with clear messages**:
   ```bash
   git commit -m "Add feature: description of what you added"
   ```
6. **Push to your fork**:
   ```bash
   git push origin feature/your-feature-name
   ```
7. **Open a Pull Request** with:
   - Description of changes
   - Why the change is needed
   - Screenshots if UI changes
   - Testing you've done

## Development Guidelines

### Code Style

- **Hugo templates**: Follow Hugo best practices
- **Markdown**: Use consistent formatting
- **YAML/TOML**: Proper indentation and syntax
- **Comments**: Explain non-obvious code

### Content Guidelines

- **Beginner-friendly**: Assume minimal technical knowledge
- **Clear explanations**: Define acronyms and technical terms
- **Practical examples**: Show real use cases
- **Encouraging tone**: Supportive, not condescending

### Testing

Before submitting a PR:

- [ ] Test locally with `hugo server -D`
- [ ] Check all links work
- [ ] Verify images load correctly
- [ ] Test on mobile viewport
- [ ] Check for typos and grammar
- [ ] Ensure builds successfully with `hugo --minify`

## Types of Contributions We're Looking For

### Documentation

- Clarifying confusing sections
- Adding troubleshooting tips
- Fixing typos and grammar
- Adding examples
- Translating to other languages

### Content

- Example blog posts
- Example projects
- Tutorial content
- Best practices guides

### Code

- Bug fixes
- Performance improvements
- Accessibility improvements
- New features (discuss in issue first)

### Design

- Theme improvements
- Mobile responsiveness
- Accessibility enhancements
- Visual polish

## What We're NOT Looking For

- Breaking changes without discussion
- Features that add significant complexity
- Opinionated styling that limits customization
- Dependencies that aren't necessary
- Changes that make it harder for beginners

## Code of Conduct

### Our Standards

- **Be respectful** - Everyone is learning
- **Be helpful** - Provide constructive feedback
- **Be patient** - Not everyone has the same experience level
- **Be inclusive** - Welcome all contributors

### Unacceptable Behavior

- Harassment or discriminatory language
- Personal attacks
- Trolling or insulting comments
- Publishing others' private information
- Other unprofessional conduct

## Questions?

Not sure if your contribution fits? Open an issue to discuss it first! We're happy to provide guidance.

## Recognition

Contributors will be:
- Listed in the README acknowledgments
- Credited in release notes
- Appreciated by students using this template!
